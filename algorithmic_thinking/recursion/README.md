# 递归

## 思想

分解问题

**“自顶向下”**：从问题的最终目标出发，将其分解为更小的、同类型的子问题（**“递”**），直到子问题简单到可以之间解决（基准情况、终止条件），然后再将结果逐个返回（**“归”**）回来。
